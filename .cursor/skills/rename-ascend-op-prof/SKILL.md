# Rename Ascend Op to Prof

## 目标

将 Ascend 自定义算子从旧名（如 `NotifyDispatch`）改为新名（如 `NotifyDispatchProf`），并保证以下层面一致：

- 对外算子名（opdef/json/注册入口）
- host + kernel + tiling 的符号名
- pybind 暴露接口与实现函数
- autogen `aclnn_*` 导出接口
- `aclnnInner*` 调用接口（会随 opdef 变化）
- 目录名/文件名/include 引用
- 用例脚本与示例调用
- 提交与推送

## 适用触发词

当用户提到以下任意场景时使用本技能：

- “算子改名”“算子重命名”“改成 *Prof”
- “同步 kernel/pybind/用例”
- “目录和文件也要重命名”
- “提交并推送这次改名”

## 执行流程

### 1) 全局盘点旧名引用

先定位大小写、下划线两种形态：

- 旧 CamelCase：如 `NotifyDispatch`
- 旧 snake_case：如 `notify_dispatch`

重点关注路径：

- `src/ascend_kernels/**`
- `src/pybind/**`
- `examples/**`
- `src/ascend_kernels/pregen/build_out/autogen/**`

### 2) 先改“符号名”，再改“文件/目录名”

先在现有文件内改名，避免重命名后漏改。

必须覆盖：

- opdef class / `OP_ADD(...)` / `IMPL_OP_OPTILING(...)`
- kernel 入口函数（如 `notify_dispatch_prof`）
- kernel 类名、tiling struct 名
- pybind 导出名（如 `notify_dispatch_prof`）和 C++ 实现函数名
- `aclnn*` 导出接口名（如 `aclnnNotifyDispatchProf*`）
- 示例脚本中的 API 调用和文档描述

### 3) 重命名目录与文件

按“旧名路径 -> 新名路径”迁移，并同步 include：

- 目录：`notify_dispatch` -> `notify_dispatch_prof`
- 文件：`notify_dispatch.*` -> `notify_dispatch_prof.*`
- json：`NotifyDispatch.json` -> `NotifyDispatchProf.json`
- 用例：`test_notify_dispatch.py` -> `test_notify_dispatch_prof.py`
- autogen：`aclnn_notify_dispatch.*` -> `aclnn_notify_dispatch_prof.*`

### 4) 修正 autogen 内部接口（关键）

`aclnn_notify_dispatch_prof.cpp` 中，inner 接口通常随 opdef 变更，需显式检查并修正：

- `aclnnInnerNotifyDispatchGetWorkspaceSize` -> `aclnnInnerNotifyDispatchProfGetWorkspaceSize`
- `aclnnInnerNotifyDispatch` -> `aclnnInnerNotifyDispatchProf`

若只改了外层 `aclnnNotifyDispatchProf*` 但没改 inner 调用，运行期会出现符号绑定不一致风险。

### 5) 回归检查

至少做以下检查：

- 旧关键字残留检查（旧类名、旧函数名、旧路径、旧脚本名）
- include 路径是否指向新文件名
- pybind API 与示例调用是否一致
- `git status` 中仅包含预期改动

### 6) 提交与推送

当用户要求提交时：

- 仅 `git add` 本次相关文件，排除无关未跟踪目录
- commit message 用一句话说明“为什么改名”
- 推送当前分支，不使用强推

## 提交信息模板

可选模板：

- `rename <OldOp> to <NewOp> across operator stack`
- `fix inner <op> prof symbol binding`

正文建议说明：

- 对齐 opdef 导致的符号变更
- 避免运行期接口不一致

## 常见遗漏清单

- 漏改 `TORCH_LIBRARY` 中的 schema 名
- 漏改示例脚本中旧 API 名
- 漏改 `aclnnInner*` 调用
- 文件改名后 include 仍指向旧头文件
- 仅改代码未改 json 的 `op` 字段
